# 🤝 Mitwirken

> **Wie du zu diesem Repository beitragen kannst**

---

## 🎉 Willkommen!

Vielen Dank für dein Interesse, zu diesem Projekt beizutragen! **Jeder Beitrag ist willkommen**, egal ob du **Fehler meldest, Dokumentation verbesserst oder neuen Code hinzufügst**. 

---

## 📌 Wie du beitragen kannst

### **🐛 Bugs melden**
- **Überprüfe bestehende Issues**: Bevor du einen neuen Bug meldest, prüfe, ob das Problem bereits in den [Issues](https://github.com/kithroetel/kithroetel/issues) gemeldet wurde.
- **Erstelle ein neues Issue**: Falls der Bug noch nicht gemeldet wurde, erstelle ein neues Issue mit:
  - Einer **klaren Beschreibung** des Problems.
  - **Schritten zur Reproduktion** (falls möglich).
  - **Screenshots oder Logs** (falls relevant).
  - Deiner **Umgebung** (z. B. Betriebssystem, Python-Version, Browser).

---

### **📖 Dokumentation verbessern**
- **Rechtschreibfehler korrigieren**: Falls du einen Tippfehler oder grammatikalischen Fehler findest, kannst du direkt einen **Pull Request** erstellen.
- **Neue Anleitungen hinzufügen**: Falls du eine **Anleitung, ein Tutorial oder eine Erklärung** hinzufügen möchtest, erstelle einen Pull Request mit deinen Änderungen.
- **Übersetzungen**: Falls du die Dokumentation in eine andere Sprache übersetzen möchtest, kontaktiere mich über ein **Issue** oder einen **Pull Request**.

---

### **💻 Code beitragen**
1. **Fork das Repository**: Klicke auf die Schaltfläche **"Fork"** oben rechts auf der Repository-Seite.
2. **Klone dein Fork**:
   ```bash
   git clone https://github.com/dein-benutzername/kithroetel.git
   cd kithroetel
   ```
3. **Erstelle einen neuen Branch**:
   ```bash
   git checkout -b feature/dein-feature
   ```
4. **Füge deine Änderungen hinzu** und commite sie:
   ```bash
   git add .
   git commit -m "Füge dein Feature hinzu"
   ```
5. **Push deine Änderungen zu deinem Fork**:
   ```bash
   git push origin feature/dein-feature
   ```
6. **Erstelle einen Pull Request**: Gehe zu deinem Fork auf GitHub und klicke auf **"Pull Request"**, um deine Änderungen vorzuschlagen.

---

### **📦 Neue Projekte oder Skripte hinzufügen**
Falls du ein **neues Projekt, Skript oder Tool** hinzufügen möchtest:
1. **Erstelle ein neues Verzeichnis** im `projects/`-Ordner (z. B. `projects/python/dein-projekt/`).
2. **Füge eine `README.md`** hinzu, die das Projekt beschreibt.
3. **Dokumentiere Abhängigkeiten** (z. B. `requirements.txt` für Python-Projekte).
4. **Füge Beispiele oder Anleitungen** hinzu, wie das Projekt verwendet wird.
5. **Erstelle einen Pull Request** mit deinen Änderungen.

---

## 📜 Richtlinien für Pull Requests

### **✅ Was wir suchen:**
- **Klarer und präziser Code**: Dein Code sollte **lesbar, gut kommentiert und gut strukturiert** sein.
- **Tests**: Falls dein Code neue Funktionen hinzufügt, füge **Tests** hinzu, um die Funktionalität zu überprüfen.
- **Dokumentation**: Aktualisiere die **Dokumentation** (z. B. `README.md`), falls deine Änderungen die Verwendung des Projekts beeinflussen.
- **Kleine, fokussierte Änderungen**: Ein Pull Request sollte **eine Sache** behandeln (z. B. einen Bug fixen oder ein Feature hinzufügen).

### **❌ Was wir vermeiden:**
- **Große, unübersichtliche Änderungen**: Teile große Änderungen in **kleinere Pull Requests** auf.
- **Ungetesteter Code**: Stelle sicher, dass dein Code **funktioniert** und keine bestehenden Tests bricht.
- **Sensible Daten**: Füge **niemals** Passwörter, API-Schlüssel oder andere sensible Daten hinzu.

---

## 🔍 Code-Review-Prozess

1. **Pull Request erstellen**: Nach dem Einreichen deines Pull Requests wird er **automatisch überprüft** (z. B. auf Link-Fehler oder Formatierungsprobleme).
2. **Review durch den Maintainer**: Ich werde deinen Pull Request **überprüfen** und ggf. **Feedback geben**.
3. **Anpassungen vornehmen**: Falls Änderungen erforderlich sind, nimm diese vor und **push die Änderungen zu deinem Branch**.
4. **Merge**: Sobald der Pull Request **genehmigt** wurde, wird er in den `main`-Branch gemergt.

## 🛠️ Entwicklungsumgebung einrichten

### **🐍 Python**
1. **Python installieren**: [Python 3.8+](https://www.python.org/downloads/)
2. **Virtual Environment erstellen**:
   ```bash
   python -m venv venv
   source venv/bin/activate  # Linux/macOS
   venv\Scripts\activate     # Windows
   ```
3. **Abhängigkeiten installieren**:
   ```bash
   pip install -r requirements.txt
   ```

### **📝 Markdown**
- Nutze einen **Markdown-Editor** wie [VS Code](https://code.visualstudio.com/) oder [Typora](https://typora.io/).
- Für **Vorschau**: GitHub zeigt Markdown-Dateien direkt an.
